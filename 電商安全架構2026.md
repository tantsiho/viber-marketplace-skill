# 電商平台安全架構總覽（2026 標準）

---

## 核心原則

- **Zero Trust**：永不信任任何人／任何請求，無論來自 client 或 server，都要驗證。
- **Defense in Depth**：多層防護（middleware → server action → database query）。
- **Least Privilege**：每個角色只給最小必要權限。
- **Server-First**：敏感邏輯永遠留在 server（Server Components / Actions），避免 client 洩漏。
- **自動化與監控**：用工具偵測、記錄、警報。

---

## 1. 認證與授權（Authentication + Authorization）

- **Auth.js v5**
  - 強制 `AUTH_SECRET` 環境變數（至少 32 字元）。
  - 啟用 HttpOnly + Secure + SameSite=Strict 的 session cookie。
  - 支援 Passkeys / WebAuthn（防釣魚，2026 主流）。
  - 避免 client-side only 檢查，永遠在 server 驗證。

- **角色與權限**
  - Prisma Role enum + middleware 粗粒度保護路由。
  - 細粒度用自訂 helper 或 Permit.io / Clerk Organizations。
  - **賣家專屬**：所有 query 強制 `where: { sellerId: session.user.sellerId }`，防 IDOR。

- **常見漏洞防範**
  - OWASP A01 Broken Access Control → middleware + server action 雙重檢查。
  - 避免 optimistic UI（client 先假設有權限），一律 server 驗證後再 render。

---

## 2. 資料保護與隱私

- **環境變數與秘密**：API key、DB URL、支付金鑰用 Vercel / `.env` 管理，絕不 commit；避免在 client bundle 出現（Next.js 會移除 server-only env）。
- **輸入驗證**：Zod + server action 驗證所有 form / API 輸入。防 SQL 注入：Prisma 參數化；自訂 raw query 用參數化。防 XSS：React 內建 escaping + CSP。
- **敏感資料**：密碼用 bcrypt / Argon2；信用卡／支付資訊絕不存 DB，用 Stripe / TapPay 等 tokenization。

---

## 3. Web 安全 Headers 與 CSP

在 `next.config.js` 或 middleware 設定：

```typescript
// next.config.js
const headers = () => [
  {
    source: '/:path*',
    headers: [
      { key: 'X-Content-Type-Options', value: 'nosniff' },
      { key: 'X-Frame-Options', value: 'DENY' },
      { key: 'Referrer-Policy', value: 'strict-origin-when-cross-origin' },
      { key: 'Strict-Transport-Security', value: 'max-age=31536000; includeSubDomains; preload' },
      {
        key: 'Content-Security-Policy',
        value: "default-src 'self'; script-src 'self' 'unsafe-eval' 'unsafe-inline' https://*.vercel-scripts.com; style-src 'self' 'unsafe-inline'; img-src 'self' data: https:; connect-src 'self' https://*.stripe.com;",
      },
    ],
  },
]
```

- CSP 嚴格版：只允許信任來源，防 XSS / 第三方 JS 注入。
- HSTS：強制 HTTPS（Vercel 預設，自訂時確認）。

---

## 4. API / Server Action 保護

- **Rate Limiting**：Upstash / Vercel KV（防 brute force、DDoS）。
- **CSRF**：Auth.js 處理 cookie-based session；form 可用 double-submit cookie 或隱藏 token。
- **CORS**：只允許信任 origin（middleware 檢查）。
- **錯誤處理**：不洩漏 stack trace / DB 錯誤到 client，回傳 generic 訊息。

---

## 5. 電商特定安全措施

- **金流**：Stripe / PayPal 等 PCI-DSS 合規服務，永不碰卡號；Webhook 驗證 signature。
- **訂單與庫存**：扣庫存用 Prisma `$transaction`，防 race condition（賣超）。
- **賣家審核**：新賣家預設 `approved = false`，admin 手動審。
- **日誌與監控**：Sentry / Logflare 記錄錯誤、異常登入、未授權嘗試。

---

## 6. 部署與 CI/CD 安全

- Vercel 自動 HTTPS + DDoS 防護。
- Dependabot / Renovate 自動更新依賴（防 supply-chain 攻擊）。
- Secrets Scanning（GitHub / Vercel 內建）。
- 定期 scan：Snyk / Corgea 掃 Next.js 專案。

---

## 7. 安全架構與防護措施（2026 生產級）

對應 OWASP Top 10 2025、PCI DSS 4.0.1、台灣個資法、供應鏈與多賣家專屬風險。達標後可接銀行金流、賣家放心入駐。

### 7.1 基礎傳輸與標頭防護（全站強制）

```typescript
// next.config.js
const securityHeaders = [
  { key: 'Strict-Transport-Security', value: 'max-age=31536000; includeSubDomains; preload' },
  { key: 'X-Content-Type-Options', value: 'nosniff' },
  { key: 'X-Frame-Options', value: 'DENY' },
  { key: 'Referrer-Policy', value: 'strict-origin-when-cross-origin' },
  { key: 'X-XSS-Protection', value: '1; mode=block' },
  { key: 'Permissions-Policy', value: 'camera=(), microphone=(), geolocation=()' },
  {
    key: 'Content-Security-Policy',
    value: `default-src 'self';
      script-src 'self' 'unsafe-inline' https://*.stripe.com https://*.google.com https://js.paypal.com;
      style-src 'self' 'unsafe-inline';
      img-src 'self' data: https: *.vercel-storage.com cdn.shopify.com;
      connect-src 'self' https://*.stripe.com https://api.paypal.com https://api.tap-pay.com.tw;
      frame-ancestors 'none';
      upgrade-insecure-requests;`
  }
]
```

- **MFA**：Admin / Seller 後台強制開啟（TOTP + WebAuthn + Passkeys）。
- **Rate Limiting + WAF**：Vercel Edge / Upstash / Cloudflare（防暴力破解、DDoS、AI 機器人）。

### 7.2 認證與授權（OWASP A01 Broken Access Control）

- Auth.js v5 + `strategy: 'jwt'` + AUTH_SECRET（32 位元以上）。
- Admin / Seller 強制 MFA。
- 賣家子帳號使用獨立 Permission 表（細粒度 RBAC）。
- 雙重檢查：middleware（粗）＋ Server Action（細）。
- 所有 Seller 查詢強制 `where: { sellerId: session.user.sellerId }`（防 IDOR）。

### 7.4 PCI DSS 4.0.1 金流安全（2026）

- 絕不儲存卡號（Stripe / TapPay / 綠界 tokenization）。
- **Requirement 11.6.1 支付頁面腳本監控**：每 7 天自動比對 hash（Stripe Radar + 自訂腳本）。
- 每季 ASV 外部掃描 + 年度 QSA 審核。
- Webhook：驗證 signature + nonce + IP 白名單 + 重放攻擊防護。
- SAQ 等級依交易量（<2 萬筆/年可用 SAQ A）。

### 7.5 台灣《個人資料保護法》合規

- 蒐集個資前明確告知（目的、類別、期間、第三人）並取得同意。
- 買家「查詢、更正、刪除、停止處理」功能（會員中心必做）。
- 個資外洩 **72 小時內** 通報個資局 + 通知當事人。
- 賣家僅能看買家必要欄位（姓名、電話、地址可遮蔽後三碼）。
- 保留「個資處理同意紀錄」至少 5 年。

### 7.6 供應鏈與第三方安全（OWASP A03）

- `npm audit --audit-level=high` 設為 CI 失敗條件；Dependabot / Renovate 自動 PR。
- 鎖定 `package-lock.json`；shadcn、Prisma 等只用官方來源。
- 賣家上傳商品圖片：掃毒（ClamAV）+ 尺寸限制 + 生成 SHA-256 hash 存 DB + 存 Cloud Storage（不直接暴露路徑）。

### 7.7 電商專屬高風險防護

| 風險類型 | 防護措施（2026） | 實作方式 |
|----------|------------------|----------|
| 刷單 / 卡盜 | 裝置指紋 + 行為分析 + 風險分數 | FingerprintJS + 自訂 rule |
| 庫存賽跑攻擊 | Prisma `$transaction` + SELECT FOR UPDATE 悲觀鎖定 | 扣庫存時用 transaction |
| 假退款 / 取消濫用 | 冷卻期 + 高風險訂單人工審核 | OrderStatus 狀態機 |
| 賣家帳號被盜 | 強制 MFA + 異地登入即時警報 | Sentry + email/SMS |
| 商品圖被 AI 替換 | 上傳後 C2PA 內容真實性標記（可選） | 防 AI 生成假圖 |
| API 濫用 | 每個 endpoint 獨立 rate limit + JWT 黑名單 | Upstash Rate Limit |
| 惡意評價 | 刷負評前須「已收貨」訂單 + 買家等級門檻 | 狀態機檢查 |
| 物流詐騙 | 出貨前驗證物流單號 + 黑名單地址 | 綠界 / 黑貓 API |
| 優惠券濫用 | 單人單碼 + 使用次數上限 + IP 綁定 | Redis 紀錄 |
| 會員資料外洩 | 敏感欄位加密儲存 + 查詢時動態解密 | Prisma middleware |

### 7.8 監控、記錄與警報

- Sentry + Logflare：記錄 403、未授權嘗試、敏感操作。
- 每日自動備份 + 異地冷備份（至少 30 天還原點）。
- 季度滲透測試（台灣資安廠商：安華、趨勢、戴夫等）。
- 資安事件應變計畫（含 72 小時通報 SOP）。

### 7.9 額外進階防護（選做）

- WAF（Cloudflare / AWS WAF）
- Bot 防護（Cloudflare Bot Management）
- 資料庫加密（at-rest + in-transit）
- 定期資安稽核報告（可公開）
- Zero Trust 架構（所有請求驗證身份）
